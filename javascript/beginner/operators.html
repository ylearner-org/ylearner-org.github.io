JavaScript Operators – Complete Guide with Examples

Operators are the symbols that tell JavaScript what to do with values — add them, compare them, assign them, combine them logically. JavaScript has a rich set of operators that go well beyond simple arithmetic, including modern additions like nullish coalescing (??) and optional chaining (?.). This lesson covers every major operator category with clear examples and a precedence reference.

⏱️ 25 min read 🎯 Beginner 📅 Updated 2026

Arithmetic Operators

Arithmetic operators perform mathematical calculations on numbers.

JavaScript
let a = 10, b = 3;

console.log(a + b);    // 13  — addition
console.log(a - b);    // 7   — subtraction
console.log(a * b);    // 30  — multiplication
console.log(a / b);    // 3.333... — division
console.log(a % b);    // 1   — remainder (modulo)
console.log(a ** b);   // 1000 — exponentiation (10³)

// Increment and decrement
let x = 5;
console.log(x++);      // 5  — post-increment: returns THEN adds
console.log(x);        // 6
console.log(++x);      // 7  — pre-increment: adds THEN returns
console.log(x--);      // 7  — post-decrement: returns THEN subtracts
console.log(x);        // 6
▶ Output
13 7 30 3.3333333333333335 1 1000 5 6 7 7 6
💡
Prefer += 1 over ++

The increment/decrement operators (++/--) have a confusing pre/post distinction. Many style guides recommend using += 1 instead for clarity — it's always obvious what it does.

Assignment Operators

Assignment operators store values in variables. The compound assignment operators combine an arithmetic operation with assignment.

JavaScript
let n = 10;

n += 5;    console.log(n);   // 15  — same as n = n + 5
n -= 3;    console.log(n);   // 12  — same as n = n - 3
n *= 2;    console.log(n);   // 24  — same as n = n * 2
n /= 4;    console.log(n);   // 6   — same as n = n / 4
n %= 4;    console.log(n);   // 2   — same as n = n % 4
n **= 3;   console.log(n);   // 8   — same as n = n ** 3

// Logical assignment (ES2021)
let user = null;
user ??= "Guest";            // assigns only if null or undefined
console.log(user);           // "Guest"
▶ Output
15 12 24 6 2 8 Guest

Comparison Operators

Comparison operators compare two values and return a boolean (true or false).

JavaScript
console.log(5 > 3);     // true
console.log(5 < 3);     // false
console.log(5 >= 5);    // true
console.log(5 <= 4);    // false

// == (loose equality) — performs type coercion
console.log(5 == "5");   // true  — string coerced to number
console.log(0 == false); // true  — false coerced to 0
console.log(null == undefined); // true

// === (strict equality) — NO type coercion
console.log(5 === "5");   // false — different types
console.log(0 === false); // false — different types
console.log(null === undefined); // false

// != and !==
console.log(5 != "5");    // false — loose: they're equal after coercion
console.log(5 !== "5");   // true  — strict: different types
▶ Output
true false true false true true true false false false false true
⚠️
Always Use === Instead of ==

Loose equality (==) applies type coercion in surprising ways. 0 == "" is true, "0" == false is true, and [] == 0 is true. These are trap cases. Use === always — it checks both value and type without any conversions.

Logical Operators

Logical operators work with boolean values, but in JavaScript they have an important extra behavior — they return one of their operands, not just true/false.

JavaScript
// && (AND) — returns first falsy value, or last value if all truthy
console.log(true && true);     // true
console.log(true && false);    // false
console.log("hello" && 42);    // 42 (both truthy, returns last)
console.log(0 && "hello");     // 0  (0 is falsy, short-circuits)

// || (OR) — returns first truthy value, or last value if all falsy
console.log(false || true);    // true
console.log(0 || "default");   // "default"
console.log("user" || "anonymous"); // "user"

// ! (NOT) — negates the boolean value
console.log(!true);            // false
console.log(!0);               // true
console.log(!"");              // true
▶ Output
true false 42 0 true default user false true true

Nullish Coalescing (??) and Optional Chaining (?.)

These modern operators (ES2020) make working with null and undefined values much safer and cleaner.

JavaScript
// ?? returns right side only if left is null or undefined
// (unlike || which triggers for ALL falsy values like 0 and "")
let count = 0;
console.log(count || 10);   // 10 — 0 is falsy, so || uses right side
console.log(count ?? 10);   // 0  — 0 is not null/undefined, so ?? keeps it

let name = null;
console.log(name ?? "Anonymous");   // "Anonymous"

// ?. (optional chaining) — safely access nested properties
const user = { profile: { name: "Alice" } };
const noUser = null;

console.log(user?.profile?.name);    // "Alice"
console.log(noUser?.profile?.name);  // undefined (no error!)

// Useful with methods too
console.log(user?.greet?.());         // undefined (no error if greet doesn't exist)
▶ Output
10 0 Anonymous Alice undefined undefined

Ternary Operator

The ternary operator is a concise way to write an if/else expression in a single line: condition ? valueIfTrue : valueIfFalse.

JavaScript
const age = 20;
const status = age >= 18 ? "adult" : "minor";
console.log(status);    // "adult"

// Nested ternary (use sparingly — can hurt readability)
const score = 75;
const grade = score >= 90 ? "A"
            : score >= 80 ? "B"
            : score >= 70 ? "C"
            : "F";
console.log(grade);     // "C"
▶ Output
adult C

When in Doubt, Use Parentheses

You don't need to memorize every precedence level. Use parentheses () to make the order of operations explicit. (a + b) * c is clearer than relying on the reader to know multiplication comes before addition.

📋 Summary

  • Arithmetic operators: +, -, *, /, %, **, ++, --
  • Always use === (strict equality) instead of == to avoid type coercion surprises.
  • && and || return operands, not just booleans — useful for short-circuit evaluation and default values.
  • ?? (nullish coalescing) is safer than || when 0 or "" are valid values.
  • ?. (optional chaining) safely accesses nested properties without throwing on null/undefined.
  • Use parentheses to make operator precedence explicit and code readable.

FAQ

What is the result of 1 + "2" in JavaScript? +

The result is the string "12". When the + operator encounters a string operand, it treats the other operand as a string too and concatenates. However, "3" - 1 gives 2 because - only does subtraction and coerces the string "3" to the number 3.

What is short-circuit evaluation? +

JavaScript stops evaluating logical expressions as soon as the outcome is determined. With &&, if the left side is falsy, the right side is never evaluated. With ||, if the left side is truthy, the right side is skipped. This is useful for conditional execution: user && sendEmail(user) only calls sendEmail if user is truthy.

What is the typeof operator and what does it return? +

typeof returns a string representing the type of its operand: "string", "number", "bigint", "boolean", "undefined", "symbol", "object", or "function". Remember: typeof null === "object" (historical bug) and typeof [] === "object" (use Array.isArray() instead).

Can I use optional chaining to call a method that might not exist? +

Yes! Use obj.method?.() — the ?. before () checks if method exists before calling it. Without optional chaining, calling a non-existent method throws TypeError: obj.method is not a function.

When should I use the ternary operator vs if/else? +

Use the ternary for simple, single-expression conditions where you're choosing between two values: const label = count === 1 ? "item" : "items". Use if/else for multi-statement blocks, side effects, or when readability suffers. Avoid deeply nested ternaries — they're notoriously hard to read.
