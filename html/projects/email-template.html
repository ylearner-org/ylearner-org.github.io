HTML Email Template Project – Marketing Email with Tables

HTML emails are unlike web pages. Email clients like Outlook, Gmail, and Apple Mail strip external stylesheets, ignore many CSS properties, and render HTML differently from browsers. The solution: table-based layouts and inline CSS — a technique that looks archaic but is the only reliable approach for cross-client email compatibility. This project builds a complete welcome email template from scratch.

ℹ️
The email client compatibility challenge

Outlook (used by millions of corporate users) renders HTML using Microsoft Word's rendering engine — not a web engine. This means many modern CSS properties simply don't work. Gmail strips <style> tags in some contexts. Testing in Litmus or Email on Acid is standard industry practice for professional email development.

Email Template Structure

We build a "Welcome to ylearner" onboarding email with these sections:

  1. Outer wrapper (background colour, centres everything)
  2. Container table (max-width 600px)
  3. Header row — brand colour background, logo/name
  4. Body row — personalised greeting, what to do next, feature highlights
  5. CTA button row — primary action
  6. Footer row — unsubscribe link, company address

Key Techniques Explained

TechniqueWhy needed
Outer wrapper table at 100% widthFills the email client window and centres the 600px container
Inner container at max 600pxStandard email width — renders well on desktop and mobile
All CSS inline on each elementGmail and some clients strip <style> blocks
VML button for OutlookOutlook ignores CSS on <a> — VML vector shapes render correctly
Conditional comments <!--[if mso]-->Targets Outlook specifically for VML fallbacks

cellpadding="0" cellspacing="0"Removes default table spacing that breaks pixel-perfect layouts

💡
Test your emails before sending

Litmus (litmus.com) and Email on Acid let you preview your email template across 100+ email clients and devices before sending. Many professional email tools (Mailchimp, Klaviyo, Campaign Monitor) also have built-in preview tools. Never send an email campaign without testing in at least Gmail, Apple Mail, and Outlook.

📋 Summary

  • HTML emails require table-based layout — Flexbox/Grid are not reliable in email clients.
  • All CSS must be inline — external stylesheets and <style> blocks are stripped by some clients.
  • Use a 600px maximum width container — the standard safe width for email.
  • Outlook requires VML for styled buttons — use conditional comments to target it.
  • Always include system font fallbacks (Arial, Helvetica) — web fonts don't load reliably in email.
  • Add <!DOCTYPE html>, charset, and the X-UA-Compatible meta for Outlook.
  • Test in real email clients before sending — use Litmus or Email on Acid.

FAQ

Can I use CSS Grid or Flexbox in emails? +

No — not reliably. Outlook (which uses the Word rendering engine) does not support Flexbox or Grid. For cross-client compatibility you must use HTML tables for layout. Some modern email clients (Apple Mail, Gmail in browser) do support Flexbox and Grid, but since Outlook is used by a large portion of business users, tables remain the safe choice.

What width should email templates be? +

600px is the industry standard. It fits comfortably on desktop email clients without horizontal scrolling, and scales down on mobile. Some modern templates use 680px. Never go above 700px — wide templates look broken on many desktop clients.

Why use VML for Outlook buttons? +

Outlook ignores background-color, padding, and border-radius on <a> tags — making CSS-styled buttons display as plain text links. VML (Vector Markup Language) is a Microsoft format supported by the Word rendering engine in Outlook. By wrapping a VML shape in Outlook conditional comments, you get a proper styled button in Outlook while other clients see the normal CSS button.
